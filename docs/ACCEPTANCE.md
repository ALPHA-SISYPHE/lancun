# 最终验收标准

## 文档状态

- 版本：0.2
- 状态：讨论中

## 1. 作业要求

- [ ] 使用 HTML、CSS、JavaScript 完成核心功能。
- [ ] 至少 3 个可正常访问的页面。
- [ ] 包含图片、表格、文字、链接和表单。
- [ ] 包含真实、可说明来源的数据显示。
- [ ] 项目为个人原创工作，引用内容均注明出处。

## 2. 内容与结构

- [ ] 用户能在短时间内理解网站主题和“展示 + 互动”的两条主线。
- [ ] 首屏以海洋美感和生命力为主要入口，不依赖大段文字或恐吓式表达。
- [ ] 页面内容围绕同一核心议题，没有无关功能。
- [ ] 从认识问题到采取行动的叙事路径完整。
- [ ] 数据、单位、年份和来源准确。

## 3. 功能

- [ ] 所有导航和页面链接正常。
- [ ] 数据能够正常加载和显示。
- [ ] 图表和表格表达一致且可读。
- [ ] 筛选、详情等交互状态正确。
- [ ] 表单具有验证、错误提示和成功反馈。
- [ ] 登录/注册使用 localStorage 模拟账户与会话；成功后右上角显示个人头像，并可进入“我的界面”。
- [ ] 登录/注册提供明确标签、密码长度验证、错误提示、成功反馈、退出登录和清除本地数据入口。
- [ ] localStorage 功能在刷新后仍能恢复预期数据。
- [ ] 页面没有阻塞使用的 JavaScript 错误。

## 4. 视觉与体验

- [ ] 颜色、字体、间距和组件符合设计系统。
- [ ] 页面具有统一且可辨识的视觉风格。
- [ ] 桌面端和移动端均无明显溢出、遮挡或错位。
- [ ] 正文清晰，颜色对比足够。
- [ ] 交互元素有悬停、焦点、点击和状态反馈。
- [ ] 动画有意义、流畅，并支持减少动态效果。
- [ ] 主视觉、排版和动效符合用户确认过的方向，不存在 AI 未经确认的最终美化。
- [ ] 图片、音视频和方向稿的使用方式符合版权和来源记录要求。
- [ ] 地图、打卡、勋章或结果动画等互动反馈清晰可理解。

## 5. 性能与稳定性

- [ ] 图片经过合理压缩并设置尺寸。
- [ ] 非首屏图片考虑延迟加载。
- [ ] 页面加载时没有明显布局跳动。
- [ ] 外部资源失败时核心内容仍可访问。
- [ ] 音视频加载失败、接口失败或实时数据不可用时有可接受的降级内容。

## 6. 提交材料

- [ ] 项目目录结构清晰。
- [ ] 运行方式说明准确。
- [ ] 功能说明完整。
- [ ] 技术路线描述准确。
- [ ] 页面截图清晰。
- [ ] 数据、图片和参考代码来源列表完整。
- [ ] 中期材料和最终实验报告与实际作品一致。
- [ ] 能说明设计方向如何由用户选择、设计系统如何提取、页面如何经过截图修复。
