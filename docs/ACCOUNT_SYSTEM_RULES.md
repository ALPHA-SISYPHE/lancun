## 1. 系统定位

本系统是澜存海洋保护网站右上角的 **「海洋守护者账户系统」**。

**它是：**

- 与全站一致的、高级沉浸的海洋保护主题账户入口
- 纪录片式叙事下的守护者身份与行动记录入口
- 科研与公益结合的语言与交互

**它不是：**

- 后台管理系统
- SaaS Dashboard
- 普通登录页
- AI Landing Page
- 默认浏览器表单套壳

登录注册弹窗不能像默认浏览器表单。  
头像菜单不能像随便生成的白色弹窗。  
整体要像「海洋守护者账户系统」，不是普通后台账号系统。

---

## 2. 全站风格必须保持

修改账户系统时必须与全站保持一致：

1. 深海背景视频  
2. 海洋纪录片质感  
3. National Geographic 式自然叙事  
4. Apple Environmental Report 式高级排版  
5. 科研与公益结合的设计语言  
6. 半透明内容层  
7. 适当留白  
8. 清晰文字层级  
9. 细线分割  
10. 少量蓝色、青绿色、琥珀色点缀  
11. 高级摄影感  
12. 页面整体高级、冷静、克制  

账户弹窗、头像菜单、状态提示均须符合上述气质，并与导航栏、背景视频协调。

---

## 3. 当前问题（待优化清单）

截至规则建立时，右上角头像存在以下问题，后续阶段须逐项对照解决：

1. 头像点击后直接进入登录弹窗，用户体验突兀  
2. 用户预期是打开个人菜单，而不是直接登录  
3. 登录框太白、太普通、太像默认表单  
4. 注册框字段过长，出现内部滚动条，很不高级 — **已针对：单页紧凑注册**
5. 登录注册没有品牌叙事，没有说明用户为什么需要账户
6. 没有个人菜单生态
7. 没有清晰的未登录 / 已登录状态区分
8. 没有为个人信息页、打卡记录、证书、志愿报名、捐款记录预留入口  
9. 注册分步流程与显示昵称字段 — **已移除：单页注册，全站以用户名展示**

---

## 4. 绝对禁止

1. 点击头像直接弹出登录框（须先打开头像菜单，再在菜单内引导登录/注册）  
2. 使用浏览器默认按钮样式  
3. 使用巨大纯白登录框  
4. 使用普通后台登录页风格  
5. 使用强阴影、大圆角、紫蓝渐变 AI 风  
6. 注册表单在桌面端出现难看的内部滚动条  
7. 把所有注册字段一次性竖向堆满  
8. 破坏现有导航栏布局  
9. 删除背景视频  
10. 影响首页 Hero 的视觉层级  
11. 让头像菜单遮挡导航文字  
12. 移动端出现横向溢出

## 5. 设计 Token（账户专用）

账户 UI 统一使用以下 token（实现时写入 `assets/css/base.css` 或账户专用 stylesheet，命名保持一致）：

| Token | 值 | 用途摘要 |
|-------|-----|----------|
| `--deep` | `#031426` | 深海底色 |
| `--deep-soft` | `rgba(3, 20, 38, 0.68)` | 半透明深底 |
| `--deep-panel` | `rgba(4, 22, 39, 0.74)` | 深色面板（菜单等） |
| `--light-panel` | `rgba(234, 245, 247, 0.70)` | 浅色半透明面板 |
| `--frost-panel` | `rgba(242, 248, 250, 0.62)` | 霜化浅色面板 |
| `--line-light` | `rgba(255, 255, 255, 0.14)` | 深底上的细线 |
| `--line-dark` | `rgba(7, 30, 48, 0.12)` | 浅底上的细线 |
| `--text-light` | `#F2F8FA` | 深底主文字 |
| `--text-muted` | `rgba(242, 248, 250, 0.68)` | 深底次要文字 |
| `--ink` | `#08233D` | 浅底主文字 |
| `--blue` | `#4DA3FF` | 主强调蓝 |
| `--cyan` | `#55D6C2` | 青绿点缀 |
| `--warning` | `#DFAE4D` | 琥珀警告/提示 |
| `--normal` | `#35AFA0` | 正常/成功态 |

**使用原则：**

- 账户弹窗可使用浅色半透明面板，但不要纯白死板  
- 菜单可使用深色半透明或浅色霜化面板，须与导航、背景视频协调  
- 全站通用 glass、brand、间距等仍遵循 [`DESIGN.md`](../DESIGN.md)；与本节冲突时，**账户组件以本节 token 为准**

## 6. 功能总目标

最终账户系统至少支持：

1. 右上角头像入口  
2. 未登录状态头像菜单  
3. 已登录状态头像菜单  
4. 登录弹窗  
5. 注册弹窗  
6. 登录 / 注册 tab 切换  
7. 单页注册（用户名 / 密码 / 身份偏好），避免长表单与分步进度条  
8. 表单校验  
9. 记住登录状态  
10. 退出登录  
11. 游客模式  
12. localStorage 模拟用户数据库  
13. 当前用户状态保存  
14. 个人信息页入口  
15. 我的环保打卡入口  
16. 我的荣誉证书入口  
17. 志愿报名记录入口  
18. 公益支持记录入口  
19. 设置入口  
20. 移动端适配  
21. 键盘可访问性  

各入口与 [`pages/profile.html`](../pages/profile.html) 侧栏仪表盘、[`pages/action.html`](../pages/action.html) 打卡/报名/捐赠数据的边界，以 [`PAGE_STRUCTURE.md`](PAGE_STRUCTURE.md) 与 [`docs/ACTION_PAGE.md`](ACTION_PAGE.md) 为准；实现时不得重复造 conflicting 数据源。

---

## 7. 分阶段执行规则

**每次只完成用户当前要求的阶段。**

- 不要顺手做下一阶段  
- 不要大规模重构无关页面  
- 不要删除已有背景视频、导航和 Footer  
- 完成后停止，并告知本阶段改了哪些文件  

后续阶段（头像菜单重构、登录弹窗视觉、分步注册、profile 入口扩展等）由用户逐次指定，未指定前不得擅自开工。
