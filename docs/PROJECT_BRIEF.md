# 项目需求简报

## 文档状态

- 版本：0.2
- 状态：讨论中
- 最近更新：2026-07-17

## 1. 作业背景

Web 编程期末个人项目。老师要求自定主题，设计一个 Web 站点。

### 基准要求

- 基于 JavaScript、HTML、CSS。
- 包含图片、表格、文字、链接和表单。
- 具有数据显示，数据可来自文件、爬虫或数据库等。
- 至少 3 个页面。
- 程序运行稳定，功能合理，界面美观并符合受众需求。
- 引用数据、图片、参考代码等必须注明出处。

### 加分方向

- 数据显示质量。
- 界面美观、主题独特。
- JavaScript 特效。
- HTML5、性能优化、安全技术或其他能够准确说明的技术。

## 2. 当前项目方向

- 主题：海洋保护宣传、科普与互动体验。
- 项目名称：澜存。
- 名称释义：留住浪潮与深海，寓意守护海洋生机。
- 暂定核心叙事：展示海洋生命的美丽与活力，同时介绍污染现状和公众可参与的保护行动。
- 当前产品结构：我们的海洋（数据看板、海的作用、五大洋；细则见 `docs/OCEAN_PAGE.md`）+ 海在呼救（污染数据、解决方案）+ 海洋生物档案（检索、识别入口）+ 保护行动中心（打卡、报名、公益项目）+ 我的界面（本地档案）。
- 内容两大分类：展示（海洋美景、海洋生物、污染现状、科普介绍、数据和音视频）与互动（地图探索、物种信息、保护行动打卡、表单记录、勋章或结果动画）。
- 已确认页面策略：方案 A 五页平衡型。
- 已确认地图策略：重点海域地图。
- 已确认行动记录策略：前端注册/登录表单 + `localStorage` 本地账户、密码、会话和行动档案，不建立真实后端账号体系。密码明文仅用于课程演示，不得使用真实密码。
- 开发性质：个人课程作品，不构建真实商业服务。

## 3. 用户分层与价值假设

### 主要用户

- 社会大众：通过美感和易懂的内容认识海洋保护议题。
- 海洋爱好者：获得更深入的物种、地图和数据探索。
- 儿童及亲子用户：通过图像、地图、轻量互动和奖励机制学习海洋知识。
- 课程作业评阅者：能够清楚看到主题价值、功能完整度和技术实现。

### 用户价值

网站帮助用户：

1. 先感受到海洋的生命美和活力，而不是一开始面对大段警示文字。
2. 通过地图、图表、图片和音视频理解海洋环境与生物。
3. 认识污染现状及其对生态系统的影响。
4. 完成一个适合自己的保护行动，并获得可视化反馈。

- [x] 行动记录策略：匿名表单与 localStorage。
- [ ] 是否加入音频、视频或自动播放的环境氛围内容。
- [ ] 互动地图的范围（全球海洋、重点海域或虚拟探索地图）。

## 5. 范围原则

- 优先完成能够覆盖评分要求且可在截止日期前稳定交付的功能。
- 不为了显得复杂而加入与主题无关的登录、支付或真实后台。
- 互动效果必须能够说明课程知识点或改善叙事体验。

## 6. 暂不纳入范围

- 真实用户账号系统。
- 真实捐款和支付（仅展示公益项目与支持意向入口）。
- 需要长期维护的线上社区。
- 未经验证的实时数据接口。
- 原生移动应用或桌面应用。
