## 1. 页面定位

- 主题线：展示「海洋之美」与正面保护 / 观测成果，服务叙事路径「欣赏海洋 → 产生好奇 → 理解作用 → 再进入污染与行动页」。
- **禁止**在本页堆叠污染灾难、恐吓式数据；污染可视化与污染源科普归 [`pages/rescue.html`](../pages/rescue.html)。
- 技术：原生 HTML5 / CSS3 / JavaScript ES6+；本页不新增后端服务。
- 视觉：服从仓库根目录 `DESIGN.md`；实现前先出结构框架稿，未经用户确认不做最终美化。
- **不改**：首页整体与「我的」界面（本宪法范围外）。首页仅允许增加链入本页锚点（如 `#ocean-dashboard`），不在本轮改首页大结构。

## 3. 三节信息架构（固定顺序）

### 3.1 第一节 — 海洋数据看板

- DOM 锚点（实现时必须提供）：`id="ocean-dashboard"`，供首页跳转。
- 目的：展示 **正面** 海洋观测与监测能力，不是污染榜。
- **布局**：仅 **6 张并列看板卡片**（桌面建议 3×2 或 2×3；移动单列或双列），分两组展示：
  - **珊瑚礁监测 · Coral Watch**（3 卡）：海表温度、热应力、白化相关
  - **沿海观测 · NOAA CO-OPS**（3 卡）：潮位、水温、风与气压
- **不得**在本节接入 Protected Planet 或额外第七张「保护区总览」卡（见 §4.1）。
- 每卡须含：指标名称、数值、单位、来源链接、加载中 / 成功 / 失败（降级）状态。
- 演示站点（实现轮默认，可写入 `mock-data` 与脚本常量）：
  - Coral Watch：`southeast_florida`（`GET https://api.coral.tsr.lol/stations/southeast_florida/current`）
  - NOAA CO-OPS：站号 `8518750`（The Battery, NY）；`date=today`；具体 `product` 见附录 A

### 3.2 第二节 — 海的作用

- 锚点建议：`id="ocean-role"`。
- 目的：说明海洋对气候调节与碳汇（碳中和叙事）的作用，用数据可视化表达。
- **指标字段与图表类型：待确认**（附录 A 第二节子表）。在确认前不得把某一套图表写成「已定稿」。

### 3.3 第三节 — 五大洋

- 锚点建议：`id="five-oceans"`。
- 竖排顺序固定：太平洋 → 大西洋 → 印度洋 → 南大洋 → 北冰洋。
- 每一块至少包含：美图（或明确占位）、详细介绍、**小数据看板**（字段待与附录 A 第三节子表或专表对齐）。
- 不要求可拖地球；五大洋选择交互以页内滚动与锚点为主。

## 4. 方案 B 数据源（强制）

### 4.1 Protected Planet API（UNEP-WCMC）— 非第一节范围

| 字段 | 内容 |
|------|------|
| 文档 | https://api.protectedplanet.net/documentation |
| 第一节 | **不接入**；6 看板仅 Coral Watch + NOAA |
| 后续扩展 | 若用户另开需求，可作保护区专题或他页数据源 |
| 密钥 | 需免费注册 API Key；**禁止**提交到 GitHub |
| 许可 | 官方声明 API 不可用于商业用途；课程作业非商业展示符合其声明方向 |

### 4.2 NOAA CO-OPS Data API（第一节 3 看板）

| 字段 | 内容 |
|------|------|
| 文档 | https://api.tidesandcurrents.noaa.gov/api/prod |
| 用途 | 第一节看板 **4–6**：潮位、水温、风速 + 气压（见附录 A） |
| 端点 | `https://api.tidesandcurrents.noaa.gov/api/prod/datagetter` |
| 密钥 | 免密钥 |
| CORS | 官方说明支持浏览器跨域；若环境仍失败则走降级 |
| 降级 | 超时 / 网络错误 → 本地 mock + 时间戳说明 |

### 4.3 Coral Watch API（第一节 3 看板）

| 字段 | 内容 |
|------|------|
| 原设想 | “Global Coral Restoration Monitor” 公开 JSON（修复面积、存活率） |
| 核验结论 | **未找到可核验的官方 REST API** |
| 锁定替代 | **Coral Watch API**：https://api.coral.tsr.lol/ |
| 用途 | 第一节看板 **1–3**：SST、热应力、白化相关（见附录 A） |
| 禁止 | 把接口响应 **伪造** 成「修复面积 / 存活率」 |
| 降级 | 失败 → 本地 mock +「演示数据」提示 |

### 4.4 技术强制条款

1. 纯前端 `fetch`；不新增后端代理（除非日后用户明确要求并改宪法）。
2. 失败（网络、CORS、无 Key、超时、非 JSON）必须降级本地 mock，并显示「当前为演示数据」和/或「上次成功时间」。
3. 实时接口 **不得** 成为作业唯一数据来源；本地降级数据必须完整可演示。
4. 所有正式数字登记到 [`DATA_SOURCES.md`](DATA_SOURCES.md)。
5. API Key 不得入库；`.gitignore` 忽略密钥文件。
