## 1. Layout law（黄金比例水平分割）

| 规则 | 要求 |
|---|---|
| 分割比 | 桌面端水平 **左 ≈ 38% / 右 ≈ 62%**（φ 近似：`1 : 1.618` → `38.2% : 61.8%`） |
| 左栏（小） | **白内容岛**承载文案 + CTA；不抢地球主体 |
| 右栏（大） | **地球作为可交互浮空物体的取景区**；不得把地球当全屏壁纸 |
| **右区中央（硬性 · v1.4）** | 「右半区」= **整屏 / section canvas 全宽** 上 `x ∈ [0.38, 1.0]`；球心投影目标 **`x = 0.69`**（`0.38 + 0.62/2`）。**禁止**以 `.ocean-explore__stage` / `page-width` 内栏中心作为球心锚点（stage 仅 CSS 占位） |
| **统一绑定锚点（硬性 · v1.7）** | 锚点：`[data-ocean-panel]` 相对 canvas-host 同帧 rect；比例常数 diameter:copyHeight=1.12（像素每帧实测，随 zoom 同步） |
| **双栏（几何判定 · v1.7）** | copy 与 stage 左右并排：球心 X=69%、Y=copyCenterY |
| **单列** | stacked：Y=canvas 50%；min(copyH×1.12, bounds-fit) |
| **Containment（硬性 · v1.8）** | 投影整球 bounds 在 **Visible Safe Rect** 内（`visualViewport ∩ canvas-host − fixed header`，≥2% 边距）；左缘 ≥ copy 右缘 + gap；**100% 浏览器 zoom 下整球必可见** |
| **Fit 优先级（v1.8）** | ① 不出画（强制）→ ② 双栏 Y 同心 → ③ 直径 copyH×1.12（尽力） |
| **X 回退（v1.8）** | 69% 放不下时，球心 X 可左移（最小：clear copy 右缘 + gap），Y 不变 |
| 实现 | 单文件 `ocean-globe.js`：`_visibleSafeRect` + `_resolveFraming`（二分 / X-nudge / forced-shrink）→ `earthGroup.position` + `camera.z`；visualViewport + RO + rAF |
| 废止 v1.9 | `assets/js/globe/**` 整栈、`setViewOffset` 构图、ScrollTrigger dolly、solver 失败仍出画 |
| 决策 | 1.12 / 锚点 A / 双栏 Y / dolly off — 2026-07-18 |

## 2. Z-layer law（深度分层 · 语义层 + Three.js 独立 render layers）

### 2.1 页面语义层（从底到顶）

| 语义层 | 内容 | 说明 |
|---|---|---|
| L0 | **通透海水蓝**背景 | Section CSS：`mist` / 浅海雾渐变（`--mist-from` → `--mist-to` 等 v4 token）；WebGL **不透明** `setClearColor` 须与该浅底一致（建议 `0xe0f2fe` / `--mist-to`）— **禁止**透明清屏导致 Chromium 白屏 |
| L1 | 部分玻璃气泡 **在地球后方** | 须能被地球遮挡或明显处于球后景深 |
| L2 | 地球（交互主体） | 离散球体；含云层、大气 rim、大陆架 overlay、`+` 标记 |
| L3 | 部分玻璃气泡 **在地球前方** | 叠在球前，液体玻璃可读（浅底上仍可见、不刺眼） |
| L4 | UI overlay | 白岛文案、大陆架 toggle、hint；`pointer-events` 见 §5 |

禁止把全部气泡画在同一层且永远盖住地球，却声称「有前后景深」。  
**临时例外（v1.9.2 · 用户批准）**：`ocean-explore--dark-interim` 允许 `#0F172A → #1E3A8A` 深底 + 浅色字，**仅用于气泡对标**；地球重做或用户撤销后须恢复 v4 mist + 白内容岛。

### 2.2 Three.js `mesh.layers` 法（硬性 · v1.2）

地球与气泡 **不得** 共享同一个 Three.js `layers` mask。它们是 **独立 render layers**；遮挡靠 **多 pass + 共享 depth buffer**（或等价正确遮挡方案），**不是**把球后气泡和地球塞进同一 `layers` 位。

| Three.js layer | 内容 | 禁止 |
|---|---|---|
| **Layer 0** | **仅**地球组：球体、大气、云层、大陆架 shelves、挂在地球上的相关 mesh | **禁止**任何气泡 mesh 使用 layer 0 |
| **Layer 1** | **仅**球后气泡（back bubbles） | 不得与地球同 mask |
| **Layer 2** | **仅**球前气泡（front bubbles） | 不得与地球同 mask |

**合法渲染示意（绑定实现意图）：**

1. Pass A（可选 refraction）：camera 只开 layer 0 → 地球写入 RT（气泡不参与）  
2. Pass B：清屏（opaque mist）→ camera 只开 **layer 1** → 画球后气泡（可写/测 depth，按材质）  
3. Pass C：`autoClear = false` → camera 只开 **layer 0** → 画地球；**depth test** 遮挡球后气泡（视觉遮挡正确）  
4. Pass D：camera 只开 **layer 2** → 画球前气泡  

等价顺序（先地球写 depth，再画 layer 1 depth-tested 气泡）亦可，只要满足：

- 球后气泡被地球 **视觉遮挡正确**  
- 气泡 mesh **从不** `layers.set(0)` / 与地球同 mask  
- 球前气泡叠在地球之上可读  

若实现上「独立 layers + 正确遮挡」无法同时满足 → **STOP 并询问用户**（给出 2 个清晰选项），禁止硬拧。

**废止（v1.2）：** 旧法「Bubbles: back（layer 0 + depthTest）/ front（layer 1）」以及「Layer 0: earth + back bubbles 同 mask」——已废止。

## 3. Earth law（地球 · v1.3 独立自转）

| 规则 | 要求 |
|---|---|
| 尺度 | **比例律**：目标 `copyHeight × 1.12`；经 **bounds-fit 闭环** 求解 z + `setViewOffset`；可缩小至 containment，不可裁切 |
| 球心 X | 双栏：整屏 **69%**（v1.4）；单列：canvas 50% 或 fit 中心 |
| **球心 Y** | 双栏：白岛 **垂直中心**（±2%）；单列：canvas **50%** |
| 稳定性 | `resize` + `visualViewport.resize` + `ResizeObserver`（copy/layout/host）；rAF 合并；**禁止** scroll dolly 改 z |
| 形态 | 清晰球体轮廓；双栏时与白岛 **同一高度线**；**整球始终在 Visible Safe Rect 内**（非仅 canvas 数学框） |
| **独立自转（硬性）** | 自动旋转 = 每帧递增 **`earthGroup.rotation.y`**（绕世界 / 铅垂 Y）；拖动 = 绕同一轴改 `rotation.y` |
| 自转轴 | **仅绕竖直 / 铅垂轴（世界 Y）**；**禁止**翻滚（tumble / 乱极角） |
| 相机 | 相机 **固定正对球心**（可微调 z）；**禁止**用 `OrbitControls.autoRotate` 或绕球转相机伪装自转 |
| 拖拽 | 自定义 pointer 拖 yaw（真正拖地球）；松手后延迟恢复自转；`prefers-reduced-motion` 时停自转，仍可拖 |
| 允许 | `+` 标记、云层、大气 rim、大陆架 mask（挂在 earthGroup，随球转） |
| 亮度 | 海洋/陆地在浅底上可读；适度提高曝光/灯光，**禁止**洗成死白 |
| 清屏 | 始终不透明浅海雾色（与语义 L0 一致），alpha `1` |

**废止（v1.3）：** 用 `OrbitControls.autoRotate` / polar lock 绕目标转相机充当「地球自转」。

## 4. Bubble law（气泡）

| 规则 | 要求 |
|---|---|
| 分布 | **全视口**（左文案区 + 右地球区），不是只挤在球旁一圈 |
| 材质 | 明显 **液体玻璃**（Apple 气质）：rim / 折射 / 通透 — **禁止**细线框空心环 |
| 强度 | 可见但克制；不压过地球与白岛（浅底上可略调 tint，仍禁过曝） |
| 动效 | 缓慢上浮 + 极轻漂移；**禁止**夸张 scale 呼吸 / 弹跳 |
| 分层 | 必须同时存在「球后」与「球前」实例；且遵守 §2.2 **独立 layers** |
| 降级 | `prefers-reduced-motion` 时停漂或静态摆放，仍须可见 |

### 10.0 整体差异总述

**目标图**：是「中心聚光 + 多层景深 + 轻盈精致」的沉浸式深海氛围，所有元素都服务于「通透、克制、有纵深感」的视觉逻辑。

**当前图**：偏向「平涂渐变 + 单层扁平 + 文字厚重」的平面化效果，缺少光影聚焦、元素体积感和空间层级。

**实现备注（用户指出）**：之前只做的是一层气泡的观感；现在代码侧虽有两层（back/front），对标时仍须按目标「多层景深」验收，不能仅以「存在两个 Three.js layer」代替目视纵深。

#### 核心差异精准定位（背景层 + 远处气泡层优先）

核心问题就是**没有分层景深**。目标站的气泡是严格的「**远 - 中 - 近**」三层结构，**每层的大小、透明度、模糊度、运动速度完全不同**，共同拼出深海空间感；而当前的所有气泡都在同一平面，背景也是平铺渐变，自然没有纵深感。

本阶段先聚焦并验收：**深色背景层（径向聚光）+ 远处气泡层（虚焦细碎）**。中景 + 近景见 §10.2.B（第二步），不废止目标。

**待优化约束**：
- **必须**达到：中心聚光、远-中-近多层景深可读（每层大小/透明度/模糊/速度可区分）、整体轻盈精致、通透克制有纵深。
- **禁止**：平涂渐变感、单层扁平斑点感、文字厚重抢戏、无光影聚焦；仅以 back/front layer 计数代替目视三层。

### 10.1 背景渐变与光影氛围

#### 目标图表现

- 采用**中心径向渐变**：视觉中心（地球所在区域 / 本阶段为地球位右区）是明亮通透的**钴蓝色**，亮度最高；向四周（尤其是四角、页面底部）逐渐过渡到**极深的藏青蓝**，有明确的「聚光感」，像深海里打向核心区域的一束光。
- 渐变过渡**极其柔和**，没有生硬的色阶分界，整体光影是「中心亮、四周暗」的包裹感，沉浸感极强。

#### 当前图表现

- 采用**从上到下的线性渐变**：整体亮度均匀，没有中心聚焦的光影逻辑，只是单纯的「上浅下深」，像平铺的背景色，没有聚光氛围。
- 蓝色调**偏灰、饱和度偏低**，整体发闷，缺少目标图那种干净通透的海水质感；底部深色过渡偏生硬，没有自然衰减的柔和感。

#### 对照表（深色背景层：径向聚光 vs 线性平铺）

| 维度 | 目标站（参考图） | 当前页面 |
|---|---|---|
| 渐变类型 | 径向渐变 `radial-gradient`，视觉中心（地球区域 / 本阶段地球位）最亮，向四周 / 四角 / 底部快速自然变暗，像深海里打了一束聚光，包裹感极强 | 线性渐变 `linear-gradient`，从上到下均匀变暗，亮度平铺，没有聚焦点，本质是「平面色块」，没有空间包裹感 |
| 色彩质感 | 中心是通透的亮钴蓝，边缘过渡到极深的藏青蓝，渐变过渡极其柔和，没有生硬色阶 | 蓝色灰度偏高，整体发闷，底部深色过渡偏硬，像两块颜色硬接 |
| 视觉作用 | 用光影天然划分出「核心内容区」和「边缘暗角」，自动引导视线聚焦 | 平铺底色，没有视觉引导，内容和背景分离感弱 |

#### 待优化约束

- **必须**：section / canvas-host 背景呈现中心（偏右核心区）通透钴蓝高亮 + 四角与底部极深藏青的**径向**聚光；过渡柔和、无生硬色阶；光影能划分核心区与边缘暗角。
- **禁止**：仅靠上浅下深线性平铺充当氛围；灰闷低饱和「发闷蓝」；底部硬切深色。

### 10.2 气泡装饰的质感与层次

**分层量化**见 **§10.2.A**（远处 · 本阶段第一步）与 **§10.2.B**（中景 + 近景 · 第二步）。以下总述仍有效，与分层法同时对照。

#### 目标图表现

- **大小层级丰富**：气泡尺寸差异极大——有直径很大的巨型半透明气泡（地球上方、右侧的大泡泡），也有米粒大小的细碎气泡，**大 / 中 / 小三级错落**，节奏非常自然。
- **真实水泡体积感**：每个气泡都有「顶部高光边缘 + 内部折射渐变」，不是纯平的色块，像真实的玻璃水泡，有通透的折射感；部分气泡带极淡的外光晕，融入背景但有体积。
- **分布有疏密逻辑**：围绕地球周围（本阶段：右区核心 / 地球位）气泡更密集，向页面边缘逐渐稀疏，有明确的视觉聚焦；同时气泡有前后遮挡关系（部分在地球前方、部分在地球后方——本阶段用远景虚化小泡 / 近景大气泡表达），强化空间深度。
- **额外细碎装饰**：气泡之间散落着极细的点状、短线状元素，模拟深海浮游碎屑，进一步提升真实感。

#### 当前图表现

- **尺寸单一扁平**：所有气泡大小接近，都是中等尺寸的圆形，没有超大 / 超小的层级区分，排列均匀呆板，没有节奏变化。
- **无体积纯平色块**：只是纯色半透明的蓝色圆形，没有高光、没有折射渐变，本质是模糊的扁平色块，完全没有水泡的通透质感，更像「背景斑点」。
- **分布无逻辑**：零散均匀分布在页面各处，没有围绕核心内容聚焦的设计；所有气泡都在同一平面，没有前后景深，和文字、背景完全扁平化叠加。（即使用 back/front 两层，目视仍可判为单层扁平。）
- **没有浮游碎屑**这类细节装饰，整体空且单调。

#### 待优化约束

- **必须**：尺寸跨度覆盖巨型半透明泡与米粒级细碎泡，大中小三级可读；具备顶部高光边缘 + 内部折射渐变 +（可选）极淡外光晕；疏密绕核心（地球位/右区），边缘更稀；远景泡更小更虚、近景泡更大更清，前后景深目视成立；增加极细点状/短线状浮游碎屑或等价细节。
- **禁止**：全体中等尺寸均质圆；纯色半透明扁平斑点（无高光/无折射）；均匀撒点无疏密；仅靠 layer 计数而无目视景深；边缘高光「描边过重 / 过曝白弧」（仍服从偏柔金标 F）。
- **形态（承接既有迭代）**：近圆球，保留可读微小液态形变；保留上浮与漂移；禁止明显尺寸脉冲与轴向 squash。

### 10.2.A 远处气泡层（本阶段第一步 · Binding）

这是当前**最缺的一层**——专门负责营造「水里氛围感」的远景气泡。目标站里这一层几乎「隐形」，但正是它让画面有了「水深」的感觉。**缺此层即判无纵深。**

本阶段与 §10.1 一并优先验收；达标前不得跳过只做中近景玻璃大泡。

#### 对照表（远处气泡层：虚焦细碎融于背景 vs 实焦扁平同平面）

| 维度 | 目标站・远处气泡层 | 当前页面（所有气泡） |
|---|---|---|
| 尺寸 | 普遍极小，多为米粒 / 绿豆大小，直径差异很小，统一偏细碎 | 尺寸都偏大，没有超细碎的远景点缀，所有气泡视觉重量接近 |
| 透明度 | 极低，**约 10%～20%**，几乎融在背景里，不仔细看察觉不到 | 透明度偏高，都很「实」，全部浮在表面，没有远近差 |
| 质感 | 加了**强模糊（blur）**，是虚焦的淡蓝色光斑，**没有任何边框、高光**，像远处水里的微小气泡 | 都是纯色实块，没有模糊虚焦感，质感扁平，像贴在背景上的斑点 |
| 运动速度 | 上浮速度**极慢**，是三层里**最慢**的，几乎肉眼难察移动 | 所有气泡速度一致，没有快慢分层 |
| 数量与分布 | 数量多、密度高，**均匀铺满整个背景**，只做氛围打底，完全不抢内容 | 数量少、分布零散，既没氛围也没重点 |

#### 待优化约束（远景层）

- **必须**：存在独立可读的远处气泡层；尺寸米粒/绿豆级细碎；透明度约 10%～20%；强虚焦 blur、淡蓝光斑、无边框无高光；上浮为三层中最慢；数量多、密铺背景打底、不抢文案与核心区。
- **禁止**：用中等实焦有高光的气泡冒充远景；远景与中近景同速同透明度；远景稀疏到看不出氛围铺底。

### 10.2.B 中景 + 近景气泡（第二步 · Binding 目标，远景达标后执行）

目标站在远景之上还有两层（用户补充；不废止，本阶段优先做完 §10.1 + §10.2.A 后再做）：

| 层 | 目标站要求 |
|---|---|
| **中景气泡** | 中等尺寸，带**轻微高光**，透明度约 **30%～50%**，上浮速度**中等**，是气泡的**主体** |
| **近景气泡** | **大尺寸**，有**清晰的高光、折射边缘**，透明度**高**，上浮速度**最快**，部分会遮挡地球位 / 文字，形成「从眼前飘过」的前后关系 |

#### 待优化约束（中近景 · 第二步）

- **必须**（在 §10.2.A 达标后）：中景与近景在尺寸、透明度、高光清晰度、上浮速度上可区分；近景可与文字/地球位产生前后遮挡感。
- **禁止**：三层气泡同一速度、同一透明度、同一清晰度；近景无高光折射却声称玻璃体积。
- **高光**：近景/中景仍服从偏柔金标 F——薄且克制，禁止描边过重 / 过曝白弧。

### 10.3 文字排版与字体层级

#### 目标图表现

- **标题极致轻盈**：参考标题（如 Where we are working）用**极细字重（thin/light）**的无衬线字体，字间距非常宽松，行高舒展，多行排列透气感极强，完全是「极简、轻盈」的现代设计感，和深海静谧氛围高度匹配。
- **正文层级清晰**：说明文字字号小、字重更轻，行高宽松，文字颜色是**低对比度的浅灰白色**，只做信息补充，完全不抢标题的视觉重心。
- **排版留白克制**：文字整体左对齐，左侧留足留白，文字块宽度紧凑，给右侧视觉主体（地球 / 本阶段地球位）留出充足空间，左右平衡感非常好，不会出现一侧过重的问题。

#### 当前图表现

- **标题厚重拥挤**：「我们在守护哪一片海？」字重偏粗（常规 / 中等），字间距紧凑，没有宽松的字距设计，字体视觉重量大，显得厚重拥挤，和通透的海洋氛围违和。
- **层级冗余杂乱**：正文上方多了一层「五大洋」小标题，额外增加了信息层级，反而让文字区显得细碎；正文字号偏小、行高偏紧，阅读呼吸感不足。
- **排版平衡感弱**：文字整体偏左、宽度偏窄，右侧大面积留白利用不当时视觉重心严重左倾，和右侧空白区域没有形成平衡（应对：右侧视为主体取景区/地球位，主动留白服务构图，而非「空着失衡」）。

#### 待优化约束

- **必须**：标题极细字重 + 宽松字距 + 舒展行高，轻盈透气；正文更小更轻、浅灰白低对比、行高宽松，不抢标题；左文块紧凑，右侧为大视觉留白（地球位）。
- **禁止**：标题偏粗拥挤、字距过紧；「五大洋」eyebrow 继续作为抢戏的冗余层级（应弱化或移除，除非用户另行批准保留）；文字区细碎堆叠导致呼吸感不足。

### 10.4 按钮样式与精致度

#### 目标图表现

- 是**极细边框的描边按钮**：边框线条纤细，圆角适中，文字用和标题统一的细体，整体修长克制，视觉重量很轻，和整体极简风格完全统一。
- 按钮内边距比例舒展，是偏修长的矩形，精致感强，没有笨重感。

#### 当前图表现

- 按钮边框更粗，文字字重偏常规，整体视觉重量大，风格偏钝、偏厚重，精致度不足。
- 按钮比例偏方正，内边距紧凑，和轻盈的页面氛围脱节。

#### 待优化约束

- **必须**：极细描边、细体字、修长比例与舒展内边距，视觉重量轻。
- **禁止**：粗边框、常规偏重字重、偏方正紧凑的笨重按钮。

### 10.5 空间景深与元素层级

#### 目标图表现

元素有明确的前后分层，z-index / 渲染层级清晰（与 §10.2.A / §10.2.B 对齐）：

1. 深色背景层（径向聚光 · §10.1）  
2. **远处气泡层**（虚焦细碎 · §10.2.A）  
3. 地球层（**本阶段：地球位留空 / 待恢复**）  
4. **中景气泡层**（主体 · §10.2.B）  
5. **近景气泡层**（大、快、可遮挡 · §10.2.B）  
6. 文字按钮层  

多层叠加形成了很强的纵深感，像真实的深海空间，元素不是贴在平面上的。每层大小、透明度、模糊度、运动速度必须不同。

#### 当前图表现

所有元素（背景、气泡、文字、按钮）都在同一平面，没有前后遮挡、没有远近景深，本质是扁平的平面排版，空间沉浸感弱。

#### 待优化约束

- **必须**：目视成立「背景 → 远景气泡 →（地球位）→ 中景气泡 → 近景气泡 → 文字按钮」；各气泡层在大小、清晰度/模糊、透明度、上浮速度上可区分。
- **禁止**：背景 / 气泡 / 文字同一平面感；仅用 DOM 叠层或仅两个 Three.js layer 计数而无目视三层差异。

### 10.6 细节与辅助元素

- **禁止**：长期缺失左下精致开关却宣称细节对齐；灰闷高灰度蓝。
