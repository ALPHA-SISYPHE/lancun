# 「我们的海洋」连续叙事版 · 本轮验收基准

## 使用范围

- 仅修改副本 2：`pages/ocean-draft2.html`、`assets/css/ocean-draft2.css`、`assets/js/ocean-draft2.js`。
- 不覆盖原页面，不修改副本 1，也不引用或执行以前任何海洋页设计文档。
- 现有 `assets/media/pages-bg.mp4` 是唯一动态视频来源：静音、循环、0.6 倍速，不替换文件。

## 核心目标

把页面从“每屏一个单一模块”的 PPT 节奏，改为在动态海洋纪录片上连续阅读的研究杂志。

页面要比副本 1 稍微更紧凑、信息更充实，但不能成为卡片墙、仪表盘或密集信息表。

## 页面结构

### 1. 首屏

- 保留现有视频封面、左侧雾化阅读区和克制的引言。
- 首屏下方不再放页内导航。

### 2. 海洋正在如何运转

- 合并原“观测现场”和“海洋作用”，控制在约一至一屏半的连续版面。
- 使用视频作为深色环境底层。
- 一个版面内同时出现：
  - “约 26%”宏观结论；
  - 2 × 3 六项公开观测；
  - 一条 CO₂ 趋势线；
  - 三条海洋作用短事实；
  - 数据意义、来源、刷新、实时 / 本地演示状态与数据说明。
- 信息通过网格、细分隔线、大数字、图表和短注释混排；不得拆成两张大表或两段同构数据区。

### 3. 五大洋

- 五大洋标题前加入低对比的大幅海洋影像作为章节引子，避免浅色纯底直接开始。
- 五章不是重复卡片：
  - 太平洋：左文右图；
  - 大西洋：左图右文；
  - 印度洋：横向大画面，信息带置于下方；
  - 南大洋：深色全幅影像叠字；
  - 北冰洋：冰雾影像与留白收尾。
- 每章保留：编号、中英文名称、短叙事、统一的三项指标、影像来源、档案弹窗。
- 每章额外增加一句不超过 18 字的“阅读旁注”，用于解释该海域的独特意义。

### 4. 底部页内导航

- 不移动全站主导航。
- 设置固定在浏览器底部的本页导航：`01 海洋运转 / 02 五大洋档案 / 03 继续探索`。
- 首屏时隐藏，离开首屏后出现；当前章节高亮；弹窗开启时隐藏。
- 深蓝低对比底、细分隔线、小号文字；移动端适配安全边距。

### 5. 结尾

- 使用深蓝动态视频承接“海在呼救”“生物档案”两个跨页出口。

## 视觉与内容约束

- 不使用整段纯色大背景、Bento、白色卡片墙、厚玻璃拟态、霓虹、气泡、波浪 SVG。
- 图片和视频面积应显著大于纯 UI 面积。
- 收紧垂直间距约 20%–30%，但大数字、图片和章节转换处保留呼吸。
- 不再让每个章节都使用“标题 + 单一内容 + 大片留白”的相同结构。
- 动效仅限视频循环、底部导航状态、按钮和键盘焦点反馈；不添加视差、数字跳动或漂浮装饰。
