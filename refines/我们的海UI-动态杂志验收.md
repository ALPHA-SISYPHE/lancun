# 「我们的海洋」动态杂志版 · 本轮验收基准

## 使用范围

- 仅适用于 `pages/ocean-draft.html`、`assets/css/ocean-draft.css`、`assets/js/ocean-draft.js`。
- 本文档不覆盖、不修改、也不依赖项目内任何旧版海洋页设计文档。
- 原 `pages/ocean.html` 及其 CSS、JS 必须保持不变。

## 核心定义

> 整页是在动态海洋纪录片上阅读的一本研究杂志，不是放在浅蓝背景上的数据网站。

页面底层由用户现有的海洋视频构成；文字、数据与影像档案以清晰、克制的编辑排版叠加其上。视频保持静音、循环、0.6 倍速，禁止替换视频文件。

## 必须实现的画面逻辑

### 1. 首屏：动态纪录片封面

- 原视频作为首屏右侧主视觉，不能替换为静态图。
- 左侧不是纯白硬切块，而是低对比的雾化阅读区；视频画面向左有自然过渡。
- 标题左对齐、尺寸大、说明文字少。
- 仅保留一个克制状态信息与“向下阅读”入口。

### 2. 观测现场：悬浮在海面上的观测记录

- 视频延续为本节底层环境，不出现浅蓝大色块 + 白色数据盒的组合。
- 数据放在两条横向观测带中：珊瑚礁监测、沿海观测。
- 六项数据采用细分割线、大数字、小状态点，不使用独立卡片、仪表盘圆环或图标堆叠。
- 保留：刷新、最后更新时间、加载状态、实时/API 失败后的本地演示状态、数据说明弹窗与来源链接。
- 内容上方有足够深的摄影遮罩，保证可读性；遮罩服务于阅读，不做玻璃卡堆叠。

### 3. 海洋作用：下潜后的信息叙事

- 使用同一原视频的深色裁切作为动态底层环境。
- 左下/左侧呈现“约 26%”主结论；右侧为无白底容器的 CO₂ 趋势线。
- 三条事实沿底部以细线和短文本呈现：热量缓冲、氧气来源、蓝碳固碳。
- 保留趋势图 / 数据表切换与来源链接；图表数据点须可键盘聚焦。

### 4. 五大洋：五个章节镜头

- 五大洋不是五张重复卡片，而是五段近一屏的影像章节。
- 太平洋：文字左、图像右；大西洋反向；印度洋采用横向大画面；南大洋为深色全幅镜头；北冰洋为近白冰雾结尾。
- 每章仅保留：`OCEAN 0x`、中英文名称、一段叙事、三项数据、影像来源、查看完整档案。
- “查看完整档案”使用弹窗；内容可为明确标识的短档案摘要，不能伪造未登记数据。

### 5. 页面结尾

- 用深蓝动态海面环境承接两个跨页出口：海在呼救、生物档案。
- 不使用普通浅色页脚直接截断叙事。

## 视觉验收规则

- 不使用纯色大区块的硬拼接。
- 不使用 Bento、浮动白卡墙、厚重玻璃拟态、霓虹、气泡、波浪 SVG 分割线或复杂地图。
- 每个屏幕只有一个视觉中心。
- 数据通过对齐、字号、细线和摄影遮罩建立层级，而不是靠卡片。
- 全站基础颜色仍使用项目既有 token；深蓝仅用于视频可读性遮罩、文字和局部强调。
- 图片/视频总体视觉面积应明显大于纯 UI 面积。
- 动效仅限：0.6 倍速视频循环、遮罩的静态层次、按钮/焦点反馈；不加入数字跳动、气泡漂浮、视差或波浪转场。

## 功能验收清单

- [ ] 视频在首屏、观测现场、海洋作用中作为动态环境播放，静音、循环、0.6 倍速。
- [ ] 视频加载失败时，内容依然可阅读。
- [ ] 六项观测数据仍可请求 API，并在失败时降级到本地 mock。
- [ ] 刷新、状态提示、数据说明弹窗可用。
- [ ] 趋势图和数据表可切换。
- [ ] 五大洋索引可跳转；五个详情弹窗可打开/关闭。
- [ ] 桌面端与移动端无横向溢出。
- [ ] 原 `ocean.html` 未被改动。
