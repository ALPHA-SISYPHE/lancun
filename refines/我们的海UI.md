## 二、详细页面结构

### 01. 页面导语：一本“数字海洋报告”的封面

不再使用现在的通用深色横幅。

桌面端采用左右不对称构图：

- 左侧约 42%
  - `01 / 我们的海洋`
  - 主标题：`地球的蓝色系统，始终在呼吸`
  - 约两行引言
  - “数据更新状态”
  - 向下阅读按钮
- 右侧约 58%
  - 一张清澈、有生命力的海洋航拍或水下照片
  - 图片角落标注地点、摄影来源
  - 叠加很小的经纬度或海域标签

主标题不居中，图片不装进普通圆角卡片。导航栏维持现有品牌样式。

导语下方增加一条章节导航：

```
观测现场  /  海洋作用  /  五大洋档案
```

桌面端滚动时可固定在导航栏下方；静态阶段先完成其视觉和锚点跳转。

### 02. 观测现场：当前海洋正在发生什么

现有 6 张数据卡内容保留，但取消“3 张 Coral Watch 卡片＋3 张 NOAA 卡片”的普通宫格。

改为一张完整的“观测台”：

```
观测现场                         数据更新时间 / 刷新
──────────────────────────────────────
珊瑚礁监测                沿海观测站
海表温度  热应力  白化指数   潮位  水温  风与气压
──────────────────────────────────────
站点说明 / 实时或演示状态 / 数据来源
```

视觉处理：

- 六项数据处于同一个平面
- 使用细分割线划分，而不是六张悬浮卡
- 数值明显大于单位和标题
- Coral Watch 与 NOAA 用文字分组，不用不同颜色卡片
- 实时数据使用一个很小的绿色状态点
- 演示数据明确显示“当前为本地演示数据”
- 不使用发光、仪表盘圆环和装饰性图标

建议补充的功能：

- “刷新数据”按钮
- 最后更新时间
- 数据加载状态
- API 失败时的演示数据提示
- 点击“数据说明”打开弹窗，解释站点、指标和数据口径
- 每组保留来源链接，但不让六个蓝色链接干扰阅读

这样既保留课程作业需要的 API 和 JavaScript，也更像真实的数据产品。

移动端变成两组纵向数据表，而不是六张单列卡片。

### 03. 海洋作用：用一个核心故事解释四组数字

现在的“四张卡＋一张折线图”缺少主次。

新版采用左右分栏：

- 左侧约 35%
  - 标题：`海洋如何维持一颗宜居星球`
  - 简短说明
  - 一个主数据：`约 26%`
  - 解释：海洋吸收的人为 CO₂ 比例
- 右侧约 65%
  - CO₂ 吸收趋势折线图
  - 图表标题、年份、单位和来源
  - 图下简短结论

其余三项数据排列成横向事实清单：

```
01  热量缓冲      约 90%
02  氧气来源      约 50%
03  蓝碳固碳      0.8–1.x Gt/年
```

每项包含：

- 编号
- 指标名称
- 大数字
- 一句话解释
- 来源按钮

建议补充功能：

- 图表数据点支持键盘聚焦
- 点击数据来源统一打开“来源与口径”弹窗
- 可切换“趋势图 / 数据表”

“数据表”可以先以普通隐藏区域或弹窗实现，这对无障碍和课程表格要求也有帮助。

### 05. 五大洋档案：五个不同节奏的影像章节

不再把五大洋放在五张相同的白卡里。

每个大洋接近一屏，统一信息内容，但构图交替：

#### 太平洋

- 左侧文字，右侧大型珊瑚或岛链图片
- 作为五大洋第一章，版面最开阔

#### 大西洋

- 图片在左，文字在右
- 使用洋流、鲸类或海岸影像

#### 印度洋

- 大图横跨页面上半部分
- 文字和三项数据放在图片下方

#### 南大洋

- 冷色全宽影像
- 大量留白
- 内容集中在左下区域

#### 北冰洋

- 近白色冰雾背景
- 深蓝文字
- 作为整页安静的结尾

每章固定包含：

- 中文名与英文名
- 章节编号，如 `OCEAN 01`
- 介绍文字
- 面积占比
- 平均深度
- 生态亮点
- 图片来源
- “查看完整档案”按钮

点击“查看完整档案”先打开弹窗，弹窗可包含：

- 地理范围
- 主要洋流
- 代表生态系统
- 典型物种
- 数据来源

这些补充内容可以先使用已有资料或短占位，不必现在增加复杂子页面。

## 四、视觉控制

为避免再次出现“设计图很好、网页很普通”的问题，这次参考图和代码采用同一套硬性规格：

- 版心最大宽度：1200px
- 标题统一左对齐
- 每个屏幕只允许一个视觉中心
- 不使用三列圆角卡片墙
- 一节最多一种主要容器
- 圆角主要用于图片、弹窗和少量控件
- 数据区域主要靠分割线、字号和对齐建立层级
- 白色与浅雾蓝交替，深蓝只用于文字和局部强调
- 图片约占整页视觉面积的 40%–50%
- 不加入气泡、漂浮光斑、水波动画等装饰；这些留到动效阶段
- 移动端按照阅读顺序重新排列，不只是简单缩小桌面布局
